# No TODOs or Partial Implementations

- Every code change must be fully functional. No `// TODO`, `// FIXME`, `// HACK`, or placeholder comments.
- No stub functions that return hardcoded values "for now".
- No partial implementations with the intent to "finish later" — either implement the feature completely or do not start it.
- If a feature is too large to implement in one pass, break it into independently complete increments. Each increment must work on its own.
